## Screenshot Best Practices

### Technical Quality
- Use high resolution (1920x1080 or higher)
- Ensure text is readable
- No pixelation or blur
- Proper lighting (if screen recording)

### Content Quality
- Use realistic, impressive examples
- Show actual functionality, not mockups
- Include diverse use cases
- Demonstrate key features

### Presentation
- Clean browser window (close unnecessary tabs)
- Hide personal information
- Use professional example data
- Consistent UI state across screenshots

## Tools for Screenshots

### macOS
- Cmd + Shift + 4: Area selection
- Cmd + Shift + 3: Full screen
- Preview: For annotations

### Windows
- Windows + Shift + S: Snipping tool
- Print Screen: Full screen
- Snip & Sketch: Annotations

### Linux
- Gnome Screenshot
- Flameshot
- Spectacle (KDE)

### Screen Recording
- OBS Studio (all platforms)
- QuickTime (macOS)
- Windows Game Bar (Windows)
- SimpleScreenRecorder (Linux)

## Annotation Tips

### What to Highlight
- Key features being demonstrated
- Important UI elements
- Data flow or process steps
- Results or outputs

### How to Annotate
- Use arrows to point to features
- Add text boxes for explanations
- Circle important elements
- Use consistent colors

### Tools
- Skitch (macOS)
- Greenshot (Windows)
- Flameshot (Linux)
- Figma (all platforms)

## Devpost Image Requirements

- **Format:** PNG or JPG
- **Size:** Max 5MB per image
- **Dimensions:** At least 1280x720
- **Quantity:** 4-8 images recommended
- **First image:** Should be the most impressive/representative

## Creating a Cover Image

**Recommended dimensions:** 1920x1080

**Should include:**
- Project name: "Nova Research Agent"
- Tagline: "AI-powered research in seconds"
- Amazon Nova logo/branding
- Key visual (screenshot or graphic)
- Clean, professional design

**Tools:**
- Canva (easy, templates available)
- Figma (professional, flexible)
- Photoshop (advanced)
- GIMP (free alternative)

## Video Hosting

### Recommended Platforms
1. **YouTube** (most common)
   - Unlisted or public
   - Good quality
   - Easy embedding

2. **Vimeo** (professional)
   - Higher quality
   - Clean interface
   - No ads

3. **Loom** (quick demos)
   - Easy recording
   - Good for short demos
   - Direct sharing

### Video Requirements
- **Length:** 2-4 minutes ideal
- **Format:** MP4, MOV, or YouTube link
- **Quality:** 1080p minimum
- **Audio:** Clear narration
- **Captions:** Recommended for accessibility
